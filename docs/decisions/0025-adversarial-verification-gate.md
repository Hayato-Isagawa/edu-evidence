# 0025. content-reviewer の後段に独立反証ゲートを追加(2 段階 → 3 段階レビュー)

- 状態: 採用
- 日付: 2026-06-26
- 関連 PR: #324
- 関連 ADR: 0003(2 段階コンテンツレビュー)

## 背景

ADR 0003 で「執筆前 verifier + PR 前 reviewer」の 2 段階レビューを確立した。しかし数値・効果量・出典帰属の検証は `edu-content-reviewer` の **単一パス** に依存しており、以下の事故がそのパスをすり抜けた:

- 教科担任制 RCT の著者・ディスカッションペーパー番号の連鎖的誤記(PR #136)
- King et al. の著者・誌名混同(*Behavior Modification* ↔ *Behavioral Disorders*)
- Finland PISA スコアの誤訂正 → 再訂正(507 → 503 → 507、PR #51)

いずれも「本文の記載を正しいと仮定して読むと見抜けない」種類の誤りで、同一レビュアーが一度通したものを同一視点で見直しても拾いにくい。

折しも Sakana AI の Conductor / TRINITY(ICLR 2026)が、**独立した検証者を多重化する(adversarial verification)** ことで成果の正確性が上がることを実証した。効果の中核は「1 パス目の見落としと相関しない独立パス」にある。これを公開前検証に取り込み、最も事故が起きた「数値・帰属」を構造的に守る。

## 検討した選択肢

- **A) `edu-content-reviewer` に観点を追加**(単一パスの強化)
- **B) 独立した反証エージェントを後段に追加**(別パス)
- **C) 重要主張を複数エージェントで並列検証し多数決**(Workflow)
- **D) 現状維持**(単一パスのまま)

## 決定

**B)** を採用。`edu-content-reviewer` の後段に、独立した反証ゲート `edu-adversarial-verifier` を 3 段階目として配置する。

A) は同一エージェント・同一コンテキスト内のため独立性が弱く、相関した見落としが残る。C) は論文に最も忠実だが、本リポジトリは Workflow 運用が未導入で飛躍が大きく、コストも増える。B) は「別エージェント = 別の初期コンテキストで主張をゼロから引き直す」ことで相関を下げつつ、運用は 1 ステップ増で済む。完全独立は同一モデルゆえ達成できないが、B) は **低リスクで取れる最大の独立性** と判断した。

### `edu-adversarial-verifier`(公開前・独立反証ゲート、`.claude/agents/`)

- **適用範囲**: 数値・効果量・一次出典への帰属を含むコンテンツに限定(全戦略 + 数値/出典を伴うコラム)。対象外(エッセイ型コラム)は `SKIP`
- **3 観点**: ①数値・効果量の独立再取得(原典の生テキストで逐語照合)②帰属の独立再照合(誌名混同・DP 番号取り違えを疑う)③反証の能動的探索(別メタ分析での効果量の揺れ・追試失敗、Rule 1.8 の根拠づけ)
- **独立性の原則**: `content-reviewer` の判定・運営者の結論を入力にせず、本文を正しいと仮定せず、原典のみを根拠にする
- **出力語彙を分離**: 主張ごとに CONFIRMED / CHALLENGED / UNVERIFIABLE、総合 CLEAR / HOLD(reviewer の PASS/WARN/BLOCK と別語彙)
- **権限**: Read / Grep / Glob / Bash / WebFetch / WebSearch(**Edit / Write を持たない**)

実行フローは `pre-write-verifier` → 執筆 → `content-reviewer` →(数値・帰属を含むもののみ)`adversarial-verifier`。

## 帰結

### 良い帰結

- 数値・帰属のすり抜け事故(PR #136 / #51 相当)を、相関しない独立パスで構造的に拾える
- 観点・出力語彙を `content-reviewer` と分けたため、役割が独立し重複しない
- 対象を「数値・帰属を含むもの」に限定したため、コストは重要コンテンツのみ 1 ステップ増

### トレードオフ

- 重要コンテンツの公開前に 1 ステップ増える(各実行は数分)
- 同一モデルの別インスタンスであり、系統的な知識の穴は共有しうる(完全独立ではない)
- WebSearch と原典取得の精度に依存する

## 撤回 / 再検討の条件

- 重要コンテンツでも CHALLENGED がほぼ出ず、過剰と判明した場合(数ヶ月運用後に再評価)
- より高い独立性が必要になった場合は、複数モデルでの並列検証・多数決(選択肢 C)への移行を検討
- Claude Code から主戦場が移った場合
