# 0008. 直線 3px アクセントバーは角張ったリスト要素のみに適用する

- 状態: 採用
- 日付: 2026-04-29
- 関連 PR: #129(本 ADR と同一 PR で確定)、関連: #122(StrategyRow 効果量バー), #125(FV 編集ピックアップで一度導入後に削除), #127(/concerns カードでは採用せず)

## 背景

サイト全体のリンク要素には主に以下の hover アクセント表現がある:

- **左 3px 緑アクセントバー**(`scaleY` で上から下へ伸びる装飾)
- **border 全周を緑に変える + タイトル色を緑に変える + 背景反転**

PR #122 で `StrategyRow`(トップページの戦略一覧)に左 3px バーを導入したのを起点として、似た UI コンポーネントで同じ装飾を再利用する場面が複数回発生している。直近の例:

- PR #125: 「悩みから探す」FV 編集ピックアップカード(rounded-xl)に一度導入したが、ユーザーレビューで「角丸カードに直線バーは合わない」として削除し、`border + タイトル緑` に統一
- PR #127: `/concerns` の悩みカード(rounded-xl)では最初から導入せず
- PR #129: `RelatedStrategyCard`(rounded-xl)で再度導入してしまい、同じ理由で削除

同じ判断が複数 PR で繰り返されているため、適用範囲のルールを明文化する。

## 検討した選択肢

### A) 全リンク要素で左 3px バーを採用する

- 利点: 視覚的アクセントが統一される
- 欠点:
  - `rounded-xl` のカードでは、直線バーが角丸の内側に収まらず語彙が破綻する
  - カード上下端でバーが切れて見え、「読み込み途中」のような不安定な印象になる
  - `overflow-hidden` で切り取れば見た目は整うが、本来の「縦に伸びるアクセント」のメタファーが弱まる

### B) 角張ったリスト要素にだけ採用する(採用)

- 利点:
  - 直線バーは矩形のリスト要素(border-t のみで rounded なし)に自然に馴染む
  - 角丸カードは `border-color + text-color + background` の変化だけで十分にホバー感を伝えられる
  - 視覚言語が明確に二系統に分かれ、コンポーネントごとに迷わない
- 欠点: 統一感はやや下がる(ただし役割の違いを示唆できる利点と相殺)

## 決定

直線 3px アクセントバーは **角張ったリスト要素にのみ** 適用する。

### 適用してよい(角張ったリスト要素)

- `StrategyRow.astro` — 戦略一覧、`border-t` のみで矩形
- 検索結果カード(Pagefind UI 内)— 同じく矩形
- そのほか `border-t` / `border-b` だけで境界を引く水平リスト

### 適用しない(角丸カード)

- `rounded-xl` / `rounded-lg` などで囲まれたカード
- 具体例:
  - FV 編集ピックアップ(`/` ホーム、PR #125)
  - `/concerns` 悩みカード(PR #127)
  - `RelatedStrategyCard.astro`(PR #129)
  - コラム最新更新カード(`/` ホーム)
  - フッター内のリンク群

### 角丸カードで使う hover 語彙

- `hover:border-[var(--color-accent)]` — 全周 border を緑に
- `group-hover:text-[var(--color-accent)]` — 主要見出し(h3 等)を緑に
- 必要に応じて `hover:bg-[var(--color-card)]` — 背景反転
- transition は `motion-safe:transition-colors motion-safe:duration-200 motion-safe:ease-out`

## 影響

- 新規カードコンポーネントを作る際は本 ADR を参照し、装飾選定の迷いをなくす
- 既存の `StrategyRow` / 検索結果カードは変更不要
- `RelatedStrategyCard`(PR #129)で導入したバーは本 ADR と同じ PR 内で削除済み
