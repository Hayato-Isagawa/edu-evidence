# 0004. 公開氏名表記を Isagawa Hayato に統一

- 状態: 採用
- 日付: 2026-04-25
- 関連 PR: #107(edu-evidence), edu-watch #11

## 背景

公開ドキュメント(JSON-LD の `author.name`、README、CLAUDE.md、PRD、エージェント定義、ソーシャル表示名など)に、運営者の氏名が日本語フルネーム(漢字表記)と英語表記の両方が混在していた。PR 作業の過程で以下が顕在化した:

- JSON-LD の `author.name` が日本語フルネームのままだった
- 複数のエージェント定義ファイル内で「運営者(〜)」の呼称に日本語フルネームが残っていた
- 修正 PR 本文の Before / After 比較表で、Before 側に日本語フルネームを書き、結果として PR description そのものに氏名が露出した

Public リポジトリの PR 本文は検索エンジンからもクロールされ、誰でも閲覧可能。漢字フルネームの露出は意図せず個人情報の公開に近づく。また学術系の出典慣習(Google Scholar など)では family-name-first が一般的で、GitHub ハンドル `Hayato-Isagawa` との整合も取りやすい。

## 検討した選択肢

- **A) 公開文書は `Isagawa Hayato` で統一、日本語フルネームは私的領域(個人メール / メモリ / 私信)に限定**
- **B) 日本語フルネームのまま続ける(日本のサイトなので)**
- **C) 匿名運営に切り替える(ハンドル名のみ)**

## 決定

**A)** を採用。

対象範囲:

- JSON-LD などの構造化データ
- README / CLAUDE.md / PRD / BRAND / CONTENT_GUIDELINES
- エージェント定義ファイルの記述
- PR タイトル / 本文(Before / After 比較表を書く場合は、Before 値を「日本語表記」「日本語フルネーム表記」等に抽象化)
- リポジトリ内コメント
- ソーシャル表示名(将来追加する際)

対象外(そのまま日本語 OK):

- 個人メール / 私的メモ / メモリ(リポジトリ外)
- コミットメッセージの履歴(force push は force risk が高いため、過去分は遡及修正しない)

## 帰結

### 良い帰結

- 公開文書から漢字フルネームが削除され、検索エンジン経由の氏名露出を最小化
- GitHub ハンドルとの綴りが完全一致し、プロフィール照合が容易
- 学術系引用(将来的な研究者へのリーチも含む)で family-name-first 形式にそのまま使える
- PR 本文の Before / After 比較表で氏名を書かないスタイルが定着

### トレードオフ

- 日本語話者の読者にとっては、運営者が誰かの直感的理解がやや減る(About ページで日本語併記は許容しうる、未決定)
- 既存のコミットメッセージに残った漢字氏名は遡及修正しないため、`git log` をたどれば履歴は残る

## 撤回 / 再検討の条件

- 一般社団法人化などで法的に本名併記が必要になった場合
- 運営者本人が日本語併記を希望する状況に変わった場合(読者層の要望など)
