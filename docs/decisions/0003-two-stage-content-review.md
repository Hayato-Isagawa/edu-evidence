# 0003. 執筆前 verifier + PR 前 reviewer による 2 段階コンテンツレビュー

- 状態: 採用
- 日付: 2026-04-25
- 関連 PR: #105, #106

## 背景

運営者は単独で執筆・レビュー・公開を担っている。コラム / 戦略ページは一次研究の数値や引用を扱うため、レビュー品質を落とせない。一方で運営者のレビュー時間は有限で、毎回すべての観点を網羅するには疲弊が大きい。

特に、PR #102 で「YouTube で d=1.20 が拡散している」を前提にした反論コラムを書いた後、WebSearch で検証した結果その前提(日本語圏で d=1.20 が拡散している)を裏付けるソースが 0 件であると判明し、コラム自体を差し替える事態が起きた(PR #104 で解説型に再生)。執筆後のレビューだけでは「前提が虚構」の事故を防げなかった。

## 検討した選択肢

- **A) 既存の汎用 `code-reviewer` 系エージェントを流用**
- **B) 執筆後の 1 段階だけにカスタムエージェントを配置**
- **C) 執筆前と執筆後の 2 段階にカスタムエージェントを分けて配置**
- **D) レビュー作業自体を運営者が毎回手動で行う(現状維持)**

## 決定

**C)** を採用。執筆前・執筆後の 2 段階を別エージェントとして切り出す。

### `edu-pre-write-verifier`(執筆前ゲート、`.claude/agents/`)

- 入力: 書こうとしているテーマと前提
- 検証: 前提事実を WebSearch で最低 5 件のソースで確認、一次ソース候補の提示、Rule 1.2a(未読文献分離)/ Rule 1.6(学校と家庭の役割境界)の抵触可能性の洗い出し、内部リンク候補の提示
- 判定: `GO` / `REVISE` / `STOP`
- 権限: Read / Grep / Glob / Bash / WebFetch / WebSearch(**Edit / Write を持たない**)

### `edu-content-reviewer`(執筆後・PR 前レビュー、`.claude/agents/`)

- 入力: 作成 / 編集した markdown ファイル
- 検証: Rule 1.1(正確性)/ 1.2a / 1.6 / 1.8(断定回避)/ 4(Public 文書スタイル)、参考資料 URL の生存、相互リンクの実在、数値 / 専門用語の密度、textlint ルール違反の予測
- 権限: Read / Grep / Glob / Bash / WebFetch / WebSearch(**Edit / Write を持たない**)

両者とも **読み専用**。修正は運営者が手動で行う設計にした。

## 帰結

### 良い帰結

- 「前提そのものが虚構」型の事故(PR #102 相当)を執筆前にブロックできる
- 運営者のレビュー時間を「機械でチェックできない判断」に集中できる
- Edit / Write を持たせないので、エージェントがゴーストライティングする懸念がない
- `CONTRIBUTING.md` の「エージェント活用早見表」で運用が明文化されている

### トレードオフ

- 新規コラムを書くたびに 2 回エージェントを走らせる手間が増える(ただし各実行は数分)
- WebSearch の精度に依存する(0 件ヒットでも STOP 判定が過剰な場合はある)
- Claude Code に依存した運用で、他のエディタ / 執筆環境に移行するとフローが途切れる

### 姉妹サイトへの展開方針

edu-watch では「前提検証」の必要度が時事ダイジェスト段階にずれ込むため、現時点ではコピーしない。Sprint 4 で週次ダイジェスト運用が始まるタイミングで、同思想の `edu-watch-digest-editor` を別途配置する。

## 撤回 / 再検討の条件

- Claude Code から他のツールに主戦場が移った場合
- エージェントの出力精度が期待値を下回ると判明した場合
- 運営者のレビュー負荷が実際には増える結果に終わった場合(3 ヶ月運用後に再評価)
